Meet App

In this project, I aimed to develop a serverless, progressive web application (PWA) using React and a test-driven development (TDD) approach. The application leverages the Google Calendar API to fetch upcoming events, combining the benefits of serverless architecture and PWAs. The key features include filtering events by city, showing/hiding event details, specifying the number of events, offline functionality, and charts for data visualization.

My Role & Key Contributions

I was responsible for designing and implementing the entire app, from the front-end development with React to the integration of serverless functions and back-end logic using AWS Lambda. I focused on providing a seamless user experience with offline support, event management, and dynamic data visualization.

Key Features

  • Show/Hide Event Details**: Users can toggle event details to control the amount of information displayed for each event.
  • Specify Number of Events to View**: Users can customize the number of events displayed in the list.
  • Offline Access**: Users can view previously accessed events when offline, ensuring continuous access to data.
  • Add to Home Screen**: Users can install the app as a PWA on their home screen for easy access.
  • City Events Chart**: A data visualization chart allows users to see upcoming events categorized by city.

Technologies Used

  • React: The application was built using React, which provided a component-based architecture for creating a dynamic, user-friendly interface.
  • Google Calendar API: Used for fetching and displaying event data. OAuth2 authentication ensures secure access to calendar data.
  • AWS Lambda: Serverless functions to manage backend operations without the need for traditional server management, allowing for scalable and cost-effective execution.
  • Service Workers: Implemented to provide offline access and caching for improved user experience.
  • Jest: Used for unit and integration testing to ensure the reliability of the app's components.
  • Jest Cucumber: Employed for user acceptance testing, following a behavior-driven development (BDD) approach.
  • Puppeteer: Used for end-to-end testing, simulating real user interactions in a headless browser environment to validate app functionality.

Lessons Learned

This project allowed me to dive deep into serverless architecture, TDD, and the mechanics of building a PWA. I gained practical experience in integrating third-party APIs, ensuring app reliability through rigorous testing, and providing offline functionality for a smoother user experience.

Challenges Overcome

Implementing offline functionality posed a challenge in terms of ensuring the app's usability when the user is disconnected. I solved this by effectively using service workers and caching strategies. Additionally, testing the app's features required a detailed approach to ensure complete test coverage, which I achieved using Jest and Puppeteer.

Future Improvements

In future iterations, I would explore adding personalized event recommendations based on user preferences and past event interactions. I would also implement more advanced data visualization techniques and expand support for additional calendar integrations.
