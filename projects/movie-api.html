Filmsphere API

Project Overview

This repository contains the server-side code for the FilmSphere web application. The application provides users with information about movies, directors, and genres. It also allows users to sign up, update personal information, and create a list of their favorite movies.

My Role and Tasks

I developed the entire server-side component of the application, including setting up the REST API, handling the database using MongoDB, and implementing business logic with Mongoose. One of the main challenges was implementing secure authentication and authorization processes.

Decisions and Outcomes

I chose to use Node.js and Express for their efficiency in handling REST APIs and MongoDB for its flexible document structure, which fit well with the movie catalog data. This combination provided scalability but required extra work on data validation and security.

What I Would Do Differently

Looking back, I would implement more comprehensive error handling and logging for better monitoring in production environments. Additionally, integrating more robust user testing early on would have improved the user experience.

Lessons Learned

From this project, I learned the importance of building secure, scalable APIs. I also improved my understanding of data modeling with MongoDB and the role of middleware in Node.js/Express for handling tasks such as authentication and validation.

Technologies Used

  • Node.js
  • Express.js
  • MongoDB & Mongoose
  • JWT (JSON Web Tokens) for authentication
  • Heroku for deployment
